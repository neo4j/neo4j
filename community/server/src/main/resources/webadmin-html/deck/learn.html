Neo4j is a Graph Database

Store anything in it, focusing on how stuff is related.

A Graph Database

Neo4j stores data in a Property Graph.

  • A Graph records data in Nodes
  • Both Nodes and Relationships store values in Properties

You know relational...

An RDBMS uses foreign keys to indicate a relation between records. Consider the classic join-table...

  • relate a Foo to some Bars using Foo_Bar
  • add columns to the join-table
  • 2 joins to follow paths from Foo to Bar

Singing: add some data, extend the path, slow the query down.

Now consider Relationships

Knowing Relational, you can imagine transforming into a Graph.

  • each Foo and Bar becomes a Node
  • the Foo_Bar records become Relationships
  • follow Relationships instead of computing joins
  • "path" queries become 1000x faster

Query time is not a function of total data set size, just local data.

This is a Property Graph

A Property Graph is a general purpose data model for storing any domain.

  • Nodes for each entity in the domain
  • Relationships to describe how the Nodes relate
  • Properties on both to store data

In a Property Graph, Relationships are First Class Citizens.

A Graph Database

Neo4j is a proper database, carefully crafted over a dozen years with a dedication to treating data with great care.

  • create, retrieve, update, delete
  • responsible ACID transactions
  • a domain specific query language, named Cypher
  • High Availability clustering (only in Enterprise)

Neo4j is a NOSQL hipster with serious business credentials.
