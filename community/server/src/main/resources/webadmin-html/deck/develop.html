Develop a Graph

From whiteboard to world stage, Neo4j works the way you think.

There are Three APIs...

Neo4j has a trio of programmatic interfaces, focused on different levels of interaction.

  • Discover the graph with REST
  • Work with the Cypher language to work with the graph
  • Customize the database with plugins for special cases

As always, pick the right tool for the job.

Discover with REST

The REST interface is very RESTful, focused on discovery through hypermedia.

  • information about the database
  • incremental graph discovery
  • basic monitoring
  • not recommended for high-load access

The REST API is fully capable but chatty.

Work with the Cypher language

Neo4j's Cypher language is purpose built for working with graph data. Inspired by SQL syntax, embracing pattern matching for describing paths.

For example, create a node for yourself... 

  • create (me { name: "Andreas" });
    • create clause to create data
      • () parenthesis to indicate a Node
    • {} brackets to introduce Properties on the Node

Cypher, continued

Now find that Node... 

  • start andreas=node(*) where n.name! = "Andreas" return n;
    • start clause to begin a query
    • andreas=node(*) to search through all Nodes
    • where clause to constrain the results

Cypher, continued

Pattern matching is the real fun of using Cypher. Now that you can find yourself, you would find your friends like this... 

  • start andreas=node(*) match (andreas)-[:KNOWS]->(friends) where n.name! = "Andreas" return friends;
    • match clause to describe the pattern from known Nodes to found Nodes

Customize with Plugins

Cypher is an expressive and powerful language, intended to cover the vast majority of operations. When you have specialized needs, plugins allow you to "get under the hood."

  • raw access to the database
  • your choice of JVM language
  • warning: for advanced users with special needs

With great power comes great responsibility.
